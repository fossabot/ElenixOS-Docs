# ElenixOS System API

ElenixOS provides a series of system-level APIs for script runtime calls. These APIs encapsulate the core functions of the system, including application management, system services, file system, and more.

## API Namespaces

ElenixOS APIs are organized in namespaces, mainly including the following namespaces:

- **eos**: Core system API
- **eos.app**: Application management API
- **eos.fs**: File system API
- **eos.services**: System services API
- **eos.ui**: UI component API
- **lv**: LVGL graphics library API

## Core System API

### eos Namespace

#### System Information

```javascript
// Get system version
const version = eos.version;

// Get system name
const name = eos.name;
```

#### System Control

```javascript
// Reboot system
eos.reboot();

// Shutdown system
eos.shutdown();

// Get system uptime (milliseconds)
const uptime = eos.uptime();
```

## Application Management API

### eos.app Namespace

#### Application Installation and Uninstallation

```javascript
// Install application
const result = eos.app.install("/path/to/app.eapk");
eos.console.log("Installation result:", result);

// Uninstall application
const result = eos.app.uninstall("com.example.app");
eos.console.log("Uninstallation result:", result);
```

#### Application List Management

```javascript
// Get installed application count
const count = eos.app.getInstalledCount();
eos.console.log("Application count:", count);

// Get application ID
const appId = eos.app.getAppId(0);
eos.console.log("Application ID:", appId);

// Check if application exists
const exists = eos.app.contains("com.example.app");
eos.console.log("Application exists:", exists);
```

#### Application Sorting

```javascript
// Move application position
eos.app.move("com.example.app", 1);
```

## File System API

### eos.fs Namespace

#### File Operations

```javascript
// Read file
const content = eos.fs.readFile("/path/to/file.txt");
eos.console.log("File content:", content);

// Write file
const result = eos.fs.writeFile("/path/to/file.txt", "Hello, ElenixOS!");
eos.console.log("Write result:", result);
```

#### Directory Operations

```javascript
// Create directory
const result = eos.fs.mkdir("/path/to/directory");
eos.console.log("Create directory result:", result);

// Recursively create directory
const result = eos.fs.mkdirRecursive("/path/to/directory");
eos.console.log("Recursive create directory result:", result);

// Recursively delete directory
const result = eos.fs.rmRecursive("/path/to/directory");
eos.console.log("Recursive delete directory result:", result);
```

#### File and Directory Checking

```javascript
// Check if it's a directory
const isDir = eos.fs.isDir("/path/to/directory");
eos.console.log("Is directory:", isDir);

// Check if it's a file
const isFile = eos.fs.isFile("/path/to/file.txt");
eos.console.log("Is file:", isFile);
```

## System Services API

### eos.services Namespace

#### Battery Service

```javascript
// Get battery level
const batteryLevel = eos.services.battery.getLevel();
eos.console.log("Battery level:", batteryLevel);

// Get charging status
const isCharging = eos.services.battery.getCharging();
eos.console.log("Is charging:", isCharging);
```

#### Sensor Service

```javascript
// Get accelerometer data
const accel = eos.services.sensor.getAccelerometer();
eos.console.log("Acceleration:", accel.x, accel.y, accel.z);

// Get heart rate sensor data
const heartRate = eos.services.sensor.getHeartRate();
eos.console.log("Heart rate:", heartRate);
```

## Event and Status Interface

### Event Listening

```javascript
// Listen to system events
eos.on("system.event", (event) => {
    eos.console.log("System event:", event);
});

// Listen to battery status changes
eos.on("battery.change", (level, charging) => {
    eos.console.log("Battery status change:", level, charging);
});
```

## Error Handling Conventions

ElenixOS API error handling follows the following conventions:

1. **Synchronous API**: Returns error code or boolean value indicating operation result
2. **Asynchronous API**: Returns error information through callback function
3. **Exception handling**: Can use try-catch to catch exceptions during API calls

### Error Codes

| Error Code | Description |
|------------|-------------|
| 0 | Success |
| -1 | Failure |
| -2 | Invalid parameter |
| -3 | Insufficient memory |
| -4 | File not found |
| -5 | Insufficient permissions |

### Error Handling Examples

```javascript
// Synchronous API error handling
const result = eos.app.install("/path/to/app.eapk");
if (result === 0) {
    eos.console.log("Installation successful");
} else {
    eos.console.log("Installation failed, error code:", result);
}

// Asynchronous API error handling
eos.fs.readFileAsync("/path/to/file.txt", (error, content) => {
    if (error) {
        eos.console.log("Failed to read file:", error);
        return;
    }
    eos.console.log("File content:", content);
});

// Exception handling
try {
    const content = eos.fs.readFile("/path/to/file.txt");
    eos.console.log("File content:", content);
} catch (error) {
    eos.console.log("Failed to read file:", error);
}
```

## API Usage Examples

### Complete Example

```javascript
// Example: Install application and monitor installation result
function installApp() {
    eos.console.log("Starting to install application...");
    const result = eos.app.install("/path/to/app.eapk");
    if (result === 0) {
        eos.console.log("Application installed successfully");
        // Get application list after successful installation
        const count = eos.app.getInstalledCount();
        eos.console.log("Current application count:", count);
        for (let i = 0; i < count; i++) {
            const appId = eos.app.getAppId(i);
            eos.console.log("Application ID:", appId);
        }
    } else {
        eos.console.log("Application installation failed, error code:", result);
    }
}

// Example: Read configuration file
function readConfig() {
    try {
        const content = eos.fs.readFile("/path/to/config.json");
        const config = JSON.parse(content);
        eos.console.log("Configuration file content:", config);
        return config;
    } catch (error) {
        eos.console.log("Failed to read configuration file:", error);
        return null;
    }
}

// Example: Monitor battery status
function monitorBattery() {
    eos.console.log("Starting to monitor battery status...");
    eos.on("battery.change", (level, charging) => {
        eos.console.log(`Battery level: ${level}%, Charging status: ${charging ? "Charging" : "Not charging"}`);
        if (level < 20 && !charging) {
            eos.console.log("Low battery, please charge in time");
        }
    });
}

// Call example functions
installApp();
readConfig();
monitorBattery();
```

## Notes

1. **Permission management**: Some APIs require specific permissions to call, please ensure the application has the corresponding permissions
2. **Resource management**: When using file system APIs, please pay attention to resource release to avoid memory leaks
3. **Error handling**: Always handle errors that may be generated by API calls to ensure application stability
4. **Performance considerations**: On resource-constrained devices, please pay attention to the frequency and overhead of API calls

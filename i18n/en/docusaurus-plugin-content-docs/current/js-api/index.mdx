# JS API 通用使用方法

ElenaOS 提供了丰富的 JavaScript API，用于开发应用和表盘。本文档将介绍 JS API 的通用使用方法，包括基本概念、命名规范、错误处理等内容。

## 基本概念

### API 命名空间

ElenaOS 的 JavaScript API 通过以下命名空间暴露给脚本：

- `eos` - 系统 API
- `lv` - LVGL UI API

### 面向对象风格

ElenaOS 的 JavaScript API 采用面向对象的风格，使用构造函数创建对象，使用实例方法操作对象：

```javascript
// 创建一个按钮
const button = new lv.button(eos.view.active());

// 设置按钮大小和位置
button.setSize(100, 50);
button.setPos(10, 10);

// 添加标签
const label = new lv.label(button);
label.setText("Click Me");
```

### 事件处理

ElenaOS 的 JavaScript API 使用回调函数处理事件：

```javascript
// 绑定点击事件
button.addEventCb((e) => {
    eos.console.log("Button clicked!");
}, lv.EVENT_CLICKED, null);
```

## 命名规范

### 常量命名

常量使用全大写字母，单词之间用下划线分隔：

```javascript
// 事件类型
lv.EVENT_CLICKED
lv.EVENT_PRESSED

// 对象标志
lv.OBJ_FLAG_CLICKABLE
lv.OBJ_FLAG_CHECKABLE

// 对齐方式
lv.ALIGN_CENTER
lv.ALIGN_TOP_LEFT
```

### 方法命名

方法使用驼峰命名法，首字母小写：

```javascript
// 设置方法
button.setSize(width, height);
button.setPos(x, y);

// 获取方法
const width = button.getWidth();
const height = button.getHeight();
```

### 构造函数命名

构造函数使用驼峰命名法，首字母小写：

```javascript
// 创建对象
const button = new lv.button(parent);
const label = new lv.label(parent);
```

## 错误处理

### 异常捕获

使用 try-catch 捕获可能的错误：

```javascript
try {
    // 可能会抛出错误的代码
    const button = new lv.button(null);
} catch (e) {
    // 处理错误
    eos.console.error("Error creating button:", e);
}
```

### 错误类型

ElenaOS 的 JavaScript API 可能会抛出以下类型的错误：

- **TypeError** - 参数类型错误
- **ReferenceError** - 引用不存在的对象
- **RangeError** - 参数值超出范围

## 内存管理

### 对象生命周期

在 ElenaOS 中，对象的生命周期由垃圾回收器管理。当对象不再被引用时，垃圾回收器会自动回收其内存。

### 手动释放

对于一些资源密集型对象，可以手动释放以节省内存：

```javascript
// 创建对象
const canvas = new lv.canvas(parent);

// 使用对象
// ...

// 手动释放对象
canvas.delete();
```

## 异步操作

### 定时器

使用 `setTimeout` 和 `setInterval` 执行异步操作：

```javascript
// 延迟执行
setTimeout(() => {
    eos.console.log("Delayed execution");
}, 1000);

// 定期执行
const intervalId = setInterval(() => {
    eos.console.log("Periodic execution");
}, 1000);

// 取消定期执行
clearInterval(intervalId);
```

### 回调函数

使用回调函数处理异步操作的结果：

```javascript
// 读取文件
fs.readFile("/path/to/file", (err, data) => {
    if (err) {
        eos.console.error("Error reading file:", err);
        return;
    }
    eos.console.log("File content:", data);
});
```

## 调试技巧

### 日志输出

使用 `eos.console.log` 输出日志信息：

```javascript
eos.console.log("Debug information");
eos.console.error("Error information");
eos.console.warn("Warning information");
```

### 断点调试

在开发环境中，可以使用断点调试来定位问题：

```javascript
// 在需要调试的地方添加断点
debugger;

// 执行代码
const result = someFunction();
eos.console.log("Result:", result);
```

## 最佳实践

1. **代码组织**：使用模块化的方式组织代码，便于维护
2. **错误处理**：使用 try-catch 捕获可能的错误
3. **内存管理**：及时释放不再使用的对象
4. **性能优化**：避免在动画期间进行复杂计算
5. **代码风格**：遵循一致的代码风格，提高可读性

## 总结

ElenaOS 的 JavaScript API 采用面向对象的风格，使用构造函数创建对象，使用实例方法操作对象。通过了解和使用这些 API，开发者可以创建出功能丰富、性能良好的应用。

本文档介绍了 JS API 的通用使用方法，包括基本概念、命名规范、错误处理等内容。开发者应该遵循这些最佳实践，以确保代码的质量和可维护性。

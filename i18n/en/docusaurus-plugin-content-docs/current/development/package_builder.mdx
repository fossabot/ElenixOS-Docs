# 应用打包

ElenaOS 使用应用打包工具将应用或表盘的源代码、资源和配置文件打包成一个统一的安装包（.eapk 文件）。本文档将详细介绍应用打包的格式、资源组织和输出规范。

## 应用打包概念

应用打包是将应用的所有组件（脚本、资源、配置）组合成一个可安装的包文件的过程。在 ElenaOS 中，打包产物是 .eapk 文件，类似于其他平台的安装包格式。

### 打包的作用

1. **统一分发**：将应用的所有组件打包成一个文件，便于分发和安装
2. **资源管理**：管理应用的资源文件，确保资源的正确加载
3. **版本控制**：包含应用的版本信息，便于版本管理
4. **权限管理**：包含应用的权限信息，确保应用只能访问授权的资源

## 输入目录结构

应用打包工具使用特定的目录结构作为输入，确保打包过程的一致性和可靠性。

### 标准目录结构

```
app/
├── manifest.json      # 应用配置文件
├── main.js            # 应用主脚本
├── icon.bin           # 应用图标
├── assets/            # 资源目录
│   ├── images/        # 图片资源
│   ├── fonts/         # 字体资源
│   └── sounds/        # 音频资源
└── lib/               # 依赖库
```

### 目录说明

- **manifest.json**：应用的配置文件，包含应用 ID、名称、版本、类型等信息
- **main.js**：应用的主脚本文件，包含应用的界面和逻辑
- **icon.bin**：应用的图标文件，用于在应用列表中显示
- **assets/**：应用的资源目录，包含图片、字体、音频等资源
- **lib/**：应用的依赖库目录，包含第三方库和模块

## 打包产物格式

ElenaOS 的应用打包产物是 .eapk 文件，它是一个压缩文件，包含了应用的所有组件。

### .eapk 文件结构

```
app.eapk/
├── manifest.json      # 应用配置文件
├── main.js            # 应用主脚本
├── icon.bin           # 应用图标
├── assets/            # 资源目录
└── lib/               # 依赖库
```

### 打包工具

ElenaOS 使用 Python 脚本 `scripts/eos_pkg_builder.py` 进行应用打包：

```bash
python3 scripts/eos_pkg_builder.py --input app/ --output app.eapk
```

## 资源合并规则

在打包过程中，资源文件会按照一定的规则进行合并和处理：

### 资源文件处理

1. **图片资源**：支持 PNG、JPG 等格式，会被压缩处理
2. **字体资源**：支持 TTF、OTF 等格式，会被优化处理
3. **音频资源**：支持 WAV、MP3 等格式，会被压缩处理

### 依赖库处理

1. **JavaScript 模块**：会被合并和优化
2. **原生库**：会被适当处理，确保在目标平台上可用

### 配置文件处理

1. **manifest.json**：会被验证和处理，确保格式正确
2. **其他配置文件**：会被复制到打包产物中

## 发布与部署方式

### 发布流程

1. **开发应用**：开发应用的源代码和资源
2. **打包应用**：使用打包工具生成 .eapk 文件
3. **测试应用**：在测试设备上测试应用
4. **发布应用**：将应用发布到应用商店或其他分发渠道

### 部署方式

1. **本地安装**：通过文件系统将 .eapk 文件复制到设备上，然后使用应用管理 API 安装
2. **远程安装**：通过网络将 .eapk 文件下载到设备上，然后使用应用管理 API 安装
3. **OTA 更新**：通过 OTA（Over-The-Air）方式更新应用

### 部署命令

```bash
# 本地安装
eas_app_install("/path/to/app.eapk");

# 远程安装（示例）
download_file("https://example.com/app.eapk", "/tmp/app.eapk");
eas_app_install("/tmp/app.eapk");
```

## 打包工具使用

### 打包工具参数

| 参数 | 描述 | 示例 |
|------|------|------|
| `--input` | 输入目录路径 | `--input app/` |
| `--output` | 输出文件路径 | `--output app.eapk` |
| `--compress` | 压缩级别（0-9） | `--compress 9` |
| `--verbose` | 详细输出 | `--verbose` |

### 运行示例

```bash
# 基本打包
python3 scripts/eos_pkg_builder.py --input app/ --output app.eapk

# 带压缩的打包
python3 scripts/eos_pkg_builder.py --input app/ --output app.eapk --compress 9

# 详细输出
python3 scripts/eos_pkg_builder.py --input app/ --output app.eapk --verbose
```

## 应用配置文件

### manifest.json 格式

```json
{
  "id": "com.example.app",
  "name": "示例应用",
  "version": "1.0.0",
  "type": "application",
  "author": "Example Developer",
  "description": "这是一个示例应用",
  "icon": "icon.bin",
  "main": "main.js",
  "permissions": [
    "battery",
    "sensor"
  ],
  "resources": {
    "images": [
      "assets/images/logo.png"
    ],
    "fonts": [
      "assets/fonts/font.ttf"
    ]
  }
}
```

### 配置字段说明

- **id**：应用的唯一标识符，格式为 `com.example.app`
- **name**：应用的显示名称
- **version**：应用的版本号，格式为 `x.y.z`
- **type**：应用类型，`application` 或 `watchface`
- **author**：应用的开发者
- **description**：应用的描述
- **icon**：应用的图标文件
- **main**：应用的主脚本文件
- **permissions**：应用需要的权限
- **resources**：应用的资源文件

## 最佳实践

1. **目录结构**：遵循标准目录结构，确保打包工具能够正确识别和处理文件
2. **资源管理**：合理组织资源文件，避免资源文件过大
3. **配置文件**：确保 manifest.json 文件格式正确，包含所有必要的字段
4. **版本控制**：使用语义化版本号，便于版本管理和更新
5. **权限管理**：只申请应用必要的权限，避免过度权限申请

## 常见问题与解决方案

### 打包失败

**问题**：打包工具执行失败

**解决方案**：
1. 检查输入目录结构是否正确
2. 检查 manifest.json 文件格式是否正确
3. 检查资源文件是否存在

### 安装失败

**问题**：应用安装失败

**解决方案**：
1. 检查 .eapk 文件是否损坏
2. 检查应用的权限是否正确
3. 检查设备存储空间是否足够

### 运行时错误

**问题**：应用运行时错误

**解决方案**：
1. 检查应用的脚本是否有语法错误
2. 检查应用的资源文件是否正确加载
3. 检查应用的权限是否被授予

## 总结

应用打包是 ElenaOS 应用开发的重要环节，它将应用的所有组件组合成一个可安装的包文件。通过了解应用打包的格式、资源组织和输出规范，开发者可以更好地管理和分发应用。

ElenaOS 的应用打包工具提供了一种简单、高效的方式来打包应用，确保应用能够在不同设备上正确安装和运行。通过遵循最佳实践，开发者可以创建高质量的应用包，为用户提供良好的应用体验。

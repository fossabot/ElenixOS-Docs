# Binding Layer Generation

ElenixOS uses a binding layer generation tool to automatically generate binding code between JavaScript and native code, mainly including SNI (Script Native Interface) and LVGL binding layers. This document will detail the binding layer generation process, parameter configuration, and verification methods.

## Binding Layer Concept

The binding layer is a bridge connecting JavaScript scripts and native C code, allowing scripts to call native functions and native code to call script functions. ElenixOS uses SNI to implement this binding mechanism.

### Functions of Binding Layer

1. **Function Mapping**: Map JavaScript function calls to native C functions
2. **Type Conversion**: Handle conversion between JavaScript types and C types
3. **Memory Management**: Manage memory allocation and deallocation for cross-language calls
4. **Error Handling**: Handle errors in cross-language calls

## Generation Process

The binding layer generation tool reads configuration files and generates corresponding binding code. The generation process mainly includes the following steps:

### 1. Prepare Configuration Files

Create a configuration file describing the functions, types, and constants to be bound. The configuration file uses JSON format, including the following main sections:

- **functions**: Functions to be bound
- **types**: Data types to be bound
- **constants**: Constants to be bound

### 2. Run Generation Tool

Execute the binding layer generation tool with the configuration file as a parameter:

```bash
python tools/binding_generator.py config/sni_config.json
```

### 3. Generated Files

The generation tool will generate the following files:

- **Header files**: Declaration of binding functions
- **Source files**: Implementation of binding functions
- **Type definition files**: JavaScript type definitions

## Configuration File Format

### Basic Structure

```json
{
    "name": "sni_module",
    "output": "src/sni/",
    "functions": [],
    "types": [],
    "constants": []
}
```

### Function Configuration

```json
{
    "name": "eos_log_print",
    "return_type": "void",
    "parameters": [
        {"name": "level", "type": "eos_log_level_t"},
        {"name": "format", "type": "const char*"},
        {"name": "ap", "type": "va_list"}
    ],
    "js_name": "logPrint"
}
```

### Type Configuration

```json
{
    "name": "eos_log_level_t",
    "type": "enum",
    "values": [
        {"name": "EOS_LOG_LEVEL_DEBUG", "value": 0},
        {"name": "EOS_LOG_LEVEL_INFO", "value": 1},
        {"name": "EOS_LOG_LEVEL_WARN", "value": 2},
        {"name": "EOS_LOG_LEVEL_ERROR", "value": 3}
    ],
    "js_name": "LogLevel"
}
```

### Constant Configuration

```json
{
    "name": "EOS_APP_DIR",
    "type": "const char*",
    "value": "\"/apps/\""
}
```

## LVGL Binding Generation

The LVGL binding layer is a special part of the binding layer, used to expose LVGL API to JavaScript.

### LVGL Configuration File

The LVGL binding configuration file needs to describe the LVGL objects, methods, and properties to be bound:

```json
{
    "name": "lvgl",
    "output": "src/sni/lv/",
    "objects": [],
    "methods": [],
    "properties": []
}
```

### Object Configuration

```json
{
    "name": "lv_obj_t",
    "parent": "lv_obj_t",
    "js_name": "obj",
    "constructor": "lv_obj_create",
    "destructor": "lv_obj_del"
}
```

### Method Configuration

```json
{
    "name": "lv_obj_set_size",
    "object_type": "lv_obj_t",
    "return_type": "void",
    "parameters": [
        {"name": "obj", "type": "lv_obj_t*"},
        {"name": "w", "type": "lv_coord_t"},
        {"name": "h", "type": "lv_coord_t"}
    ],
    "js_name": "setSize"
}
```

## Verification Methods

After generating the binding layer code, it needs to be verified to ensure correctness.

### Compilation Verification

Compile the generated code to check for syntax errors:

```bash
make
```

### Unit Testing

Write unit tests to verify the binding functions:

```javascript
// Test binding function
const result = eos.log.print(eos.log.DEBUG, "Test message");
console.log(result);
```

### Integration Testing

Integrate the generated binding layer into the ElenixOS system for overall testing:

1. Load the binding module
2. Call binding functions in JavaScript
3. Verify the execution results

## Notes

### Type Safety

Ensure type safety when generating binding layers:

- Check parameter types
- Handle pointer operations carefully
- Avoid memory leaks

### Performance Optimization

Optimize the generated binding code:

- Reduce type conversion overhead
- Use inline functions for frequently called functions
- Cache frequently used objects

### Version Compatibility

Ensure the generated binding layer is compatible with different versions:

- Version checks when generating
- Handle API changes gracefully
- Provide backward compatibility

## Tool Usage

### Command Line Parameters

```bash
python tools/binding_generator.py [options] config.json
```

| Parameter | Description |
|-----------|-------------|
| `-o, --output` | Specify output directory |
| `-v, --verbose` | Enable verbose output |
| `-d, --dry-run` | Dry run mode (no actual generation) |
| `-h, --help` | Show help information |

#### SNI Binding Configuration Example

```json
{
    "name": "eos_sni",
    "output": "src/sni/eos/",
    "functions": [
        {
```

## Summary

The binding layer generation tool is an important part of ElenixOS development, which can automatically generate JavaScript and native code binding code, improving development efficiency and code quality. By properly configuring and using the binding layer generation tool, developers can quickly expose native functions to JavaScript, enabling flexible and powerful script development capabilities.

# Changelog

## [v0.2.0-alpha.1]

### Changed

- Refactored script interface architecture, `lv_bindings` was refactored into SNI layer.
- Refactored page management architecture, `nav`, `screen_mgr`, and `scene` were integrated into the Activity concept.
- Optimized CardPager and SlideWidget.
- Refactored language interface, `current_lang` was privatized to avoid out-of-bounds access.
- Optimized list transition animation.
- Optimized panel retraction logic for control center and message list during activity switching.
- Optimized flash display logic.
- Optimized app list animation parameters.
- Optimized null pointer check and display logic in clock update callback.
- Optimized activity switching and animation handling.
- Optimized callback removal logic for scrollable objects.
- Optimized timer auto-delete function.
- Optimized debug bar position management, added global position synchronization and recovery functions.
- Optimized scrollable object lookup logic.
- Optimized scrollbar display and hide logic.
- Optimized script path parsing and image source setting.
- Optimized general handle parsing capability.
- Optimized property setting.
- Optimized animation variable management.
- Optimized memory cleanup and color type bridge support.
- Optimized file directory structure.
- Optimized animation property configuration, custom callbacks, and JS animation path integration.
- Optimized dynamic updates of source files and header files.
- Optimized script engine initialization and exit callbacks.
- Optimized parameter checking and type conversion for LVGL object creation functions.

### Added

- Added Activity lifecycle management functionality, supporting Activity creation, destruction, pause, resume, etc.
- Added page management module, pages are uniformly managed using stack, and Activity concept is introduced.
- Added SNI module, providing clearer script interface definition, supporting object-oriented LVGL operations.
- Added SNI encapsulation for LVGL components, including:
  - image component
  - dropdown component
  - checkbox component
  - canvas component
  - button matrix component
  - calendar component
- Added Activity page switching animation support, including start and end callbacks.
- Added app header logic and title change animation support.
- Added OPEN slide panel state and improved state flow.
- Added built-in backup watch face support.
- Added module import functionality, supporting JS module import.
- Added interface to get previous Activity.
- Added system initialization error handling and watch face activity retrieval functionality.
- Added theme primary color definition.
- Added LOGO display in device information view.
- Added Toast prompt functionality for character icon colors.
- Added encoder target setting as Activity View, added slide component open and close events.
- Added state management and animation transition functionality for slide components.
- Added chart component and related APIs.
- Added image button support.
- Added script app Debug page.
- Added float type bridge support.
- Added global constant export functionality.
- Added new color types and related methods.
- Added snake_case to camelCase conversion function.
- Added alias fallback functionality to support type mapping.
- Added console logging functionality, added support for different log levels.
- Added initialization and mounting functionality for ElenaOS API.

### Fixed

- Fixed issue where watch face list couldn't slide by touching watch face images.
- Fixed memory leak issues.
- Fixed script engine issues to prevent entering error state during normal operation.
- Fixed callback management for scrollable objects, optimized scrollbar display and hide logic, resolved scrollbar flickering issues.
- Fixed null pointer check in clock update callback.
- Prevented app title bar from displaying in watch face activity.
- Fixed issue where scrollable objects were lost when control center automatically rebounded.
- Fixed issue where scrollable objects couldn't be obtained when returning to Activity.
- Fixed processing logic when root screen is empty during activity creation.
- Fixed constructor parameter type check to ensure only objects or null are accepted.
- Fixed scrollable object validity check.
- Fixed issue where method names could be empty.

## [v0.1.0-alpha.1]

### Added

- Provided basic functional framework for smartwatch system.
- Provided JavaScript script running and lifecycle management capabilities.
- Provided JavaScript API support, including:
  - Partial LVGL Widget functionality;
  - LVGL event system registration and deletion;
  - LVGL animation creation and running;
  - LVGL timer-related functionality.
- Provided virtual display for PC-side simulation and debugging.
- Provided preliminary hardware abstraction and portable interface design for multi-platform adaptation.
- Established basic structure for engineering and development documentation.
- Provided developer tool scripts for quickly generating project structure.
- Provided modular system architecture, supporting core functionality combination and extension by modules.
- Provided isolation mechanism between scripts and system core to improve system stability and security.
- Provided basic debugging and development support capabilities to improve desktop-side development and debugging efficiency.

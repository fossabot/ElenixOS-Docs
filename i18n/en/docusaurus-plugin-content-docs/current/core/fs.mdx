# File System Module

The file system module is one of the core infrastructures of ElenixOS, providing a unified file and directory operation interface. This module supports multiple underlying file system backends through a portable abstraction layer, so upper-layer applications do not need to care about specific storage implementations.

## Directory Structure

```
├── src/core/system/file_system/
│   ├── eos_fs.c          # File system advanced utility functions
│   ├── eos_fs.h          # File system utility interface definitions
│   ├── eos_dfw.c         # Deferred File Writer
│   └── eos_dfw.h         # Deferred File Writer interface definitions
├── src/port/file_system/
│   ├── eos_fs_port.h     # File system porting interface definition
│   ├── eos_fs_port_posix.c    # POSIX platform porting implementation
│   └── eos_fs_port_rtthread.c # RT-Thread platform porting implementation
```

## System Directory Structure

ElenixOS maintains a standard system directory structure in the file system:

```
<EOS_SYS_ROOT_DIR>/
├── .sys/
│   ├── config/
│   │   ├── cfg.json          # System configuration file
│   │   └── app_order.json    # Application sorting configuration
│   └── res/
│       ├── img/              # System resource images
│       └── font/             # Font files
├── app/
│   ├── apps/                 # Installed applications directory
│   │   └── <app_id>/
│   │       ├── main.js       # Application main script
│   │       ├── manifest.json # Application manifest
│   │       ├── icon.bin      # Application icon
│   │       └── assets/       # Application resources
│   └── app_data/             # Application data directory
│       └── <app_id>/
│           └── config.json   # Application configuration
└── .simulator/               # Simulator resources
    └── img/
```

## Backend Support

The file system module supports the following backends through the porting interface defined in `eos_fs_port.h`:

| Backend Type | Macro Definition | File Type | Directory Type | Description |
|-------------|-----------------|-----------|---------------|-------------|
| POSIX | `EOS_FS_POSIX` | `FILE*` | `DIR*` | Linux/macOS desktop environment |
| FATFS | `EOS_FS_FATFS` | `FIL*` | `FF_DIR*` | Embedded FAT file system |
| LittleFS | `EOS_FS_LITTLEFS` | `lfs_file_t*` | `lfs_dir_t*` | Embedded flash file system |
| RT-Thread | `EOS_FS_RTTHREAD` | `int` | `DIR*` | RT-Thread DFS |
| Custom | `EOS_FS_CUSTOM` | Custom | Custom | User-defined implementation |

## Core API

### File Operations

```c
#include "eos_fs.h"
```

#### Open File

```c
eos_file_t eos_fs_open_read(const char *path);
eos_file_t eos_fs_open_write(const char *path);
```

| Function | Description |
|----------|-------------|
| `eos_fs_open_read` | Open file in read-only mode, file must exist |
| `eos_fs_open_write` | Open file in write mode, create if not exists |

**Parameters**:
- `path`: File path

**Return value**: File handle, returns `EOS_FILE_INVALID` on failure

#### Read/Write File

```c
int eos_fs_read(eos_file_t file, void *buf, size_t len);
int eos_fs_write(eos_file_t file, const void *buf, size_t len);
```

| Function | Description |
|----------|-------------|
| `eos_fs_read` | Read data from file to buffer |
| `eos_fs_write` | Write buffer data to file |

**Parameters**:
- `file`: File handle (returned by `eos_fs_open_read` or `eos_fs_open_write`)
- `buf`: Data buffer
- `len`: Number of bytes to read/write

**Return value**: Number of bytes actually read/written, returns -1 on failure

#### Close File

```c
void eos_fs_close(eos_file_t file);
```

#### File Positioning and Size

```c
int eos_fs_seek(eos_file_t file, uint32_t pos);
int eos_fs_size(eos_file_t file, uint32_t *size);
```

| Function | Description |
|----------|-------------|
| `eos_fs_seek` | Position file read/write position to specified offset (from file start) |
| `eos_fs_size` | Get file size |

#### File Synchronization

```c
int eos_fs_sync(eos_file_t file);
```

Sync file buffer data to underlying storage device.

### Advanced File Operations

#### Write String

```c
int eos_fs_puts(const char *s, eos_file_t fp);
```

Write string to file (automatically appends newline). Returns number of bytes written, negative value on failure.

#### Read Complete File

```c
char *eos_fs_read_file(const char *path);
```

Read entire file content to dynamically allocated buffer.

**Note**: Must call `eos_free` to release memory after use.

#### Write Complete File

```c
int eos_fs_write_file(const char *path, const void *data, size_t data_size);
```

Write data to file at once. Returns number of bytes actually written, -1 on failure.

#### Create File (if not exists)

```c
int eos_create_file_if_not_exist(const char *path, const char *default_content);
```

Create file if it doesn't exist, and write default content. `default_content` can be NULL (create empty file).

### Directory Operations

#### Create Directory

```c
int eos_fs_mkdir(const char *path);
```

Create single-level directory. Returns 0 for success, -1 for failure.

#### Recursively Create Directory

```c
int eos_fs_mkdir_recursive(const char *path);
```

Recursively create directory tree, e.g., `a/b/c` will create all non-existent directories level by level.

#### Recursively Delete Directory

```c
int eos_fs_rm_recursive(const char *path);
```

Recursively delete directory and all its subfiles and subdirectories.

**Security Protection**: This function prevents deletion of root directory (`/` or `\`).

#### Create Directory (if not exists)

```c
int eos_fs_mkdir_if_not_exist(const char *path);
```

#### Delete Empty Directory

```c
int eos_fs_rmdir(const char *path);
```

#### Open/Read/Close Directory

```c
eos_dir_t eos_fs_opendir(const char *path);
int eos_fs_readdir(eos_dir_t dir, char *name, size_t max_len);
void eos_fs_closedir(eos_dir_t dir);
```

Directory traversal example:

```c
eos_dir_t dir = eos_fs_opendir("/path/to/dir");
if (dir) {
    char filename[256];
    while (eos_fs_readdir(dir, filename, sizeof(filename)) == 0) {
        // Skip . and .. 
        if (strcmp(filename, ".") == 0 || strcmp(filename, "..") == 0) {
            continue;
        }
        printf("Entry: %s\n", filename);
    }
    eos_fs_closedir(dir);
}
```

### File Status Query

```c
int eos_fs_exists(const char *path);
int eos_fs_type(const char *path);
bool eos_is_dir(const char *path);
bool eos_is_file(const char *path);
```

| Function | Description |
|----------|-------------|
| `eos_fs_exists` | Check if file or directory exists, returns 1 if exists, 0 if not, <0 on error |
| `eos_fs_type` | Get path type, returns `eos_fs_type_t` enum value |
| `eos_is_dir` | Determine if path is a directory |
| `eos_is_file` | Determine if path is a file |

### File Move/Rename

```c
int eos_fs_mv(const char *old_path, const char *new_path);
```

## Deferred File Writer (DFW)

DFW (Deferred File Writer) is an optimization component of the file system module, used to reduce wear on storage media caused by frequent file write operations.

```c
#include "eos_dfw.h"
```

### Enable DFW

Configure in `eos_config.h`:

```c
#define EOS_DFW_ENABLE 1
```

### API

```c
void eos_dfw_init(void);
bool eos_dfw_write(const char *path, uint8_t *data, size_t data_size);
uint8_t *eos_dfw_read(const char *path);
void eos_dfw_sync(void);
```

| Function | Description |
|----------|-------------|
| `eos_dfw_init` | Initialize DFW module |
| `eos_dfw_write` | Write data to buffer queue, returns true if added to queue |
| `eos_dfw_read` | Read file data (from actual file system), need to manually free returned pointer |
| `eos_dfw_sync` | Sync all pending write data in buffer to actual file system |

### Working Principle

```
Write Request
    │
    ▼
┌─────────────────┐
│  Lookup Buffer Queue  │
└──────┬──────────┘
       │
    ┌──┴──┐
    │Exists?│
    └──┬──┘
   ┌───┴───┐
   │       │
   ▼       ▼
  Write    Create New
  Buffer  Buffer
   │       │
   └───┬───┘
       ▼
  Add to Queue
       │
       ▼
  eos_dfw_sync()
       │
       ▼
  Write to File System
```

## Usage Examples

### Create File and Write

```c
// Method 1: Use advanced interface
const char *content = "Hello, ElenixOS!";
int ret = eos_fs_write_file("/path/to/file.txt", content, strlen(content));
if (ret > 0) {
    printf("Write successful, %d bytes\n", ret);
}

// Method 2: Use file handle
eos_file_t fp = eos_fs_open_write("/path/to/file.txt");
if (fp != EOS_FILE_INVALID) {
    eos_fs_puts("Hello, ElenixOS!", fp);
    eos_fs_close(fp);
}
```

### Read File

```c
char *data = eos_fs_read_file("/path/to/file.txt");
if (data) {
    printf("File content: %s\n", data);
    eos_free(data);  // Must free memory
}
```

### Recursively Create Directory

```c
int ret = eos_fs_mkdir_recursive("/data/apps/my_app/assets");
if (ret == 0) {
    printf("Directory created successfully\n");
}
```

### Recursively Delete Directory

```c
int ret = eos_fs_rm_recursive("/data/temp");
if (ret == 0) {
    printf("Directory deleted\n");
}
```

### Check Path Type

```c
if (eos_is_file("/path/to/file.txt")) {
    printf("This is a file\n");
}

if (eos_is_dir("/path/to/dir")) {
    printf("This is a directory\n");
}

int type = eos_fs_type("/path/to/unknown");
if (type == EOS_FS_TYPE_NOT_EXIST) {
    printf("Path does not exist\n");
}
```

### Use DFW for Optimized Writing

```c
#if EOS_DFW_ENABLE
// Deferred writing (data written to buffer first)
eos_dfw_write("/data/config.json", (uint8_t *)json_str, strlen(json_str));

// Sync to file system at appropriate time
eos_dfw_sync();
#endif
```

## Performance Optimization Suggestions

### Reduce Write Frequency

For frequently updated configuration files (such as user settings), it is recommended:
1. Use DFW deferred write mechanism to merge multiple writes into one
2. Trigger sync after data stabilizes, avoid writing for every small change

### Batch File Operations

```c
// Not recommended: Frequent individual calls
for (int i = 0; i < 100; i++) {
    eos_fs_write_file(path, data, size);
}

// Recommended: Merge writes
eos_file_t fp = eos_fs_open_write(path);
if (fp != EOS_FILE_INVALID) {
    for (int i = 0; i < 100; i++) {
        eos_fs_write(fp, data, size);
    }
    eos_fs_close(fp);
}
```

### Avoid Recursive Deletion of Large Directories

`eos_fs_rm_recursive` may consume more time when the directory hierarchy is very deep or the number of files is huge. It is recommended:
- Perform deletion operations when the system is idle
- Split deletion operations to avoid long-term blocking

### Memory Management

After reading files with `eos_fs_read_file`, be sure to call `eos_free` to release memory:

```c
char *data = eos_fs_read_file(path);
if (data) {
    // Use data
    process_data(data);
    // Release memory
    eos_free(data);
}
```

## Error Handling

All file operation functions follow the following error handling conventions:

- Functions returning pointers: Return `NULL` on failure
- Functions returning integers: Return negative value (-1) on failure
- File handles: Return `EOS_FILE_INVALID` on failure
- Directory handles: Return `EOS_DIR_INVALID` on failure

It is recommended to always check return values:

```c
eos_file_t fp = eos_fs_open_read("/path/to/file");
if (fp == EOS_FILE_INVALID) {
    // Handle error
    return;
}
// Normal use
eos_fs_close(fp);
```

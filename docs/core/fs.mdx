# 文件系统模块

文件系统模块是 ElenixOS 的核心基础设施之一，提供统一的文件和目录操作接口。该模块通过可移植的抽象层，支持多种底层文件系统后端，使上层应用无需关心具体的存储实现。

## 目录结构

```
├── src/core/system/file_system/
│   ├── eos_fs.c          # 文件系统高级工具函数
│   ├── eos_fs.h          # 文件系统工具接口定义
│   ├── eos_dfw.c         # 延迟文件写入器（Deferred File Writer）
│   └── eos_dfw.h         # 延迟文件写入器接口定义
├── src/port/file_system/
│   ├── eos_fs_port.h     # 文件系统移植接口定义
│   ├── eos_fs_port_posix.c    # POSIX 平台移植实现
│   └── eos_fs_port_rtthread.c # RT-Thread 平台移植实现
```

## 系统目录结构

ElenixOS 在文件系统中维护了一套标准的系统目录结构：

```
<EOS_SYS_ROOT_DIR>/
├── .sys/
│   ├── config/
│   │   ├── cfg.json          # 系统配置文件
│   │   └── app_order.json    # 应用排序配置
│   └── res/
│       ├── img/              # 系统资源图片
│       └── font/             # 字体文件
├── app/
│   ├── apps/                 # 已安装应用目录
│   │   └── <app_id>/
│   │       ├── main.js       # 应用主脚本
│   │       ├── manifest.json # 应用清单
│   │       ├── icon.bin      # 应用图标
│   │       └── assets/       # 应用资源
│   └── app_data/             # 应用数据目录
│       └── <app_id>/
│           └── config.json   # 应用配置
└── .simulator/               # 模拟器资源
    └── img/
```

## 后端支持

文件系统模块通过 `eos_fs_port.h` 定义的移植接口，支持以下后端：

| 后端类型 | 宏定义 | 文件类型 | 目录类型 | 说明 |
|----------|--------|----------|----------|------|
| POSIX | `EOS_FS_POSIX` | `FILE*` | `DIR*` | Linux/macOS 桌面环境 |
| FATFS | `EOS_FS_FATFS` | `FIL*` | `FF_DIR*` | 嵌入式 FAT 文件系统 |
| LittleFS | `EOS_FS_LITTLEFS` | `lfs_file_t*` | `lfs_dir_t*` | 嵌入式闪存文件系统 |
| RT-Thread | `EOS_FS_RTTHREAD` | `int` | `DIR*` | RT-Thread DFS |
| Custom | `EOS_FS_CUSTOM` | 自定义 | 自定义 | 用户自定义实现 |

## 核心 API

### 文件操作

```c
#include "eos_fs.h"
```

#### 打开文件

```c
eos_file_t eos_fs_open_read(const char *path);
eos_file_t eos_fs_open_write(const char *path);
```

| 函数 | 说明 |
|------|------|
| `eos_fs_open_read` | 以只读方式打开文件，文件必须存在 |
| `eos_fs_open_write` | 以写入方式打开文件，文件不存在则创建 |

**参数**：
- `path`：文件路径

**返回值**：文件句柄，失败返回 `EOS_FILE_INVALID`

#### 读写文件

```c
int eos_fs_read(eos_file_t file, void *buf, size_t len);
int eos_fs_write(eos_file_t file, const void *buf, size_t len);
```

| 函数 | 说明 |
|------|------|
| `eos_fs_read` | 从文件读取数据到缓冲区 |
| `eos_fs_write` | 将缓冲区数据写入文件 |

**参数**：
- `file`：文件句柄（由 `eos_fs_open_read` 或 `eos_fs_open_write` 返回）
- `buf`：数据缓冲区
- `len`：读取/写入的字节数

**返回值**：实际读取/写入的字节数，失败返回 -1

#### 关闭文件

```c
void eos_fs_close(eos_file_t file);
```

#### 文件定位与大小

```c
int eos_fs_seek(eos_file_t file, uint32_t pos);
int eos_fs_size(eos_file_t file, uint32_t *size);
```

| 函数 | 说明 |
|------|------|
| `eos_fs_seek` | 定位文件读写位置到指定偏移量（从文件开头） |
| `eos_fs_size` | 获取文件大小 |

#### 文件同步

```c
int eos_fs_sync(eos_file_t file);
```

将文件缓冲区数据同步到底层存储设备。

### 高级文件操作

#### 写入字符串

```c
int eos_fs_puts(const char *s, eos_file_t fp);
```

向文件中写入字符串（自动追加换行符）。返回写入的字节数，失败返回负值。

#### 读取完整文件

```c
char *eos_fs_read_file(const char *path);
```

将整个文件内容读取到动态分配的缓冲区中。

**注意**：使用后必须调用 `eos_free` 释放内存。

#### 写入完整文件

```c
int eos_fs_write_file(const char *path, const void *data, size_t data_size);
```

将数据一次性写入文件。返回实际写入的字节数，失败返回 -1。

#### 创建文件（如不存在）

```c
int eos_create_file_if_not_exist(const char *path, const char *default_content);
```

如果文件不存在则创建，并写入默认内容。`default_content` 可为 NULL（创建空文件）。

### 目录操作

#### 创建目录

```c
int eos_fs_mkdir(const char *path);
```

创建单级目录。返回 0 表示成功，-1 表示失败。

#### 递归创建目录

```c
int eos_fs_mkdir_recursive(const char *path);
```

递归创建目录树，如 `a/b/c` 将逐级创建所有不存在的目录。

#### 递归删除目录

```c
int eos_fs_rm_recursive(const char *path);
```

递归删除目录及其所有子文件和子目录。

**安全保护**：该函数会防止删除根目录（`/` 或 `\`）。

#### 创建目录（如不存在）

```c
int eos_fs_mkdir_if_not_exist(const char *path);
```

#### 删除空目录

```c
int eos_fs_rmdir(const char *path);
```

#### 打开/读取/关闭目录

```c
eos_dir_t eos_fs_opendir(const char *path);
int eos_fs_readdir(eos_dir_t dir, char *name, size_t max_len);
void eos_fs_closedir(eos_dir_t dir);
```

遍历目录示例：

```c
eos_dir_t dir = eos_fs_opendir("/path/to/dir");
if (dir) {
    char filename[256];
    while (eos_fs_readdir(dir, filename, sizeof(filename)) == 0) {
        // 跳过 . 和 .. 
        if (strcmp(filename, ".") == 0 || strcmp(filename, "..") == 0) {
            continue;
        }
        printf("Entry: %s\n", filename);
    }
    eos_fs_closedir(dir);
}
```

### 文件状态查询

```c
int eos_fs_exists(const char *path);
int eos_fs_type(const char *path);
bool eos_is_dir(const char *path);
bool eos_is_file(const char *path);
```

| 函数 | 说明 |
|------|------|
| `eos_fs_exists` | 检查文件或目录是否存在，返回 1 存在，0 不存在，\<0 错误 |
| `eos_fs_type` | 获取路径类型，返回 `eos_fs_type_t` 枚举值 |
| `eos_is_dir` | 判断路径是否为目录 |
| `eos_is_file` | 判断路径是否为文件 |

### 文件移动/重命名

```c
int eos_fs_mv(const char *old_path, const char *new_path);
```

## 延迟文件写入器 (DFW)

DFW（Deferred File Writer）是文件系统模块的优化组件，用于减少频繁的文件写入操作对存储介质的磨损。

```c
#include "eos_dfw.h"
```

### 启用 DFW

在 `eos_config.h` 中配置：

```c
#define EOS_DFW_ENABLE 1
```

### API

```c
void eos_dfw_init(void);
bool eos_dfw_write(const char *path, uint8_t *data, size_t data_size);
uint8_t *eos_dfw_read(const char *path);
void eos_dfw_sync(void);
```

| 函数 | 说明 |
|------|------|
| `eos_dfw_init` | 初始化 DFW 模块 |
| `eos_dfw_write` | 将数据写入缓冲区队列，返回 true 表示已加入队列 |
| `eos_dfw_read` | 读取文件数据（从实际文件系统），需手动释放返回的指针 |
| `eos_dfw_sync` | 将缓冲区中所有待写入数据同步到实际文件系统 |

### 工作原理

```
写入请求
    │
    ▼
┌─────────────────┐
│  查找缓冲区队列   │
└──────┬──────────┘
       │
    ┌──┴──┐
    │存在？│
    └──┬──┘
   ┌───┴───┐
   │       │
   ▼       ▼
 写入    创建新
 缓冲区  缓冲区
   │       │
   └───┬───┘
       ▼
  加入队列
       │
       ▼
  eos_dfw_sync()
       │
       ▼
  写入文件系统
```

## 使用示例

### 创建文件并写入

```c
// 方法一：使用高级接口
const char *content = "Hello, ElenixOS!";
int ret = eos_fs_write_file("/path/to/file.txt", content, strlen(content));
if (ret > 0) {
    printf("写入成功，共 %d 字节\n", ret);
}

// 方法二：使用文件句柄
eos_file_t fp = eos_fs_open_write("/path/to/file.txt");
if (fp != EOS_FILE_INVALID) {
    eos_fs_puts("Hello, ElenixOS!", fp);
    eos_fs_close(fp);
}
```

### 读取文件

```c
char *data = eos_fs_read_file("/path/to/file.txt");
if (data) {
    printf("文件内容: %s\n", data);
    eos_free(data);  // 必须释放内存
}
```

### 递归创建目录

```c
int ret = eos_fs_mkdir_recursive("/data/apps/my_app/assets");
if (ret == 0) {
    printf("目录创建成功\n");
}
```

### 递归删除目录

```c
int ret = eos_fs_rm_recursive("/data/temp");
if (ret == 0) {
    printf("目录已删除\n");
}
```

### 检查路径类型

```c
if (eos_is_file("/path/to/file.txt")) {
    printf("这是一个文件\n");
}

if (eos_is_dir("/path/to/dir")) {
    printf("这是一个目录\n");
}

int type = eos_fs_type("/path/to/unknown");
if (type == EOS_FS_TYPE_NOT_EXIST) {
    printf("路径不存在\n");
}
```

### 使用 DFW 优化写入

```c
#if EOS_DFW_ENABLE
// 延迟写入（数据先写入缓冲区）
eos_dfw_write("/data/config.json", (uint8_t *)json_str, strlen(json_str));

// 在合适的时机同步到文件系统
eos_dfw_sync();
#endif
```

## 性能优化建议

### 减少写入次数

对于频繁更新的配置文件（如用户设置），建议：
1. 使用 DFW 延迟写入机制，合并多次写入为一次
2. 在数据稳定后再触发同步，避免每次小变化都写入

### 批量文件操作

```c
// 不推荐：频繁单独调用
for (int i = 0; i < 100; i++) {
    eos_fs_write_file(path, data, size);
}

// 推荐：合并写入
eos_file_t fp = eos_fs_open_write(path);
if (fp != EOS_FILE_INVALID) {
    for (int i = 0; i < 100; i++) {
        eos_fs_write(fp, data, size);
    }
    eos_fs_close(fp);
}
```

### 避免递归删除大目录

`eos_fs_rm_recursive` 在目录层级很深或文件数量巨大时可能会消耗较多时间。建议：
- 在系统空闲时执行删除操作
- 拆分删除操作，避免长时间阻塞

### 内存管理

使用 `eos_fs_read_file` 读取文件后，务必调用 `eos_free` 释放内存：

```c
char *data = eos_fs_read_file(path);
if (data) {
    // 使用数据
    process_data(data);
    // 释放内存
    eos_free(data);
}
```

## 错误处理

所有文件操作函数均遵循以下错误处理约定：

- 返回指针的函数：失败返回 `NULL`
- 返回整数的函数：失败返回负值（-1）
- 文件句柄：失败返回 `EOS_FILE_INVALID`
- 目录句柄：失败返回 `EOS_DIR_INVALID`

建议始终检查返回值：

```c
eos_file_t fp = eos_fs_open_read("/path/to/file");
if (fp == EOS_FILE_INVALID) {
    // 处理错误
    return;
}
// 正常使用
eos_fs_close(fp);
```
